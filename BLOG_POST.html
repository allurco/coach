Coach: an AI accountability buddy that actually does the thing

A self-hosted, multi-tenant AI coach that lives between you and your goals. Built in Laravel, written for me, open-sourced because the missing piece was never the model.

The itch

I started Coach because I needed someone to ask me, every morning, "did you actually pay that bill yesterday?". My finances were tangled — overdraft, parcelamento, INSS atrasado, business and personal money living in the same checking account. I had a spreadsheet. The spreadsheet did not care if I read it.

What I wanted was a chat where I could:

  • drop a PDF of an Itaú invoice and get a structured read in 30 seconds;
  • say "vou ligar pro contador hoje" and have an action with a deadline appear in my plan;
  • get a 7am email each weekday with the one most-urgent thing, then reply to it from my phone and have the reply continue the same conversation server-side;
  • not lose context after closing the tab.

I tried ChatGPT and Claude with prompts. Both forget. Neither will email me at 7am. Both will happily lie about creating a calendar event they never created.

The shape

Coach is a Laravel 13 + Filament v5 app. The chat lives in a Filament page; Livewire 4 streams the model's tokens straight into the bubble. The agent is built on the official laravel/ai SDK with the Gemini provider — Gemini 2.5 Pro for the interactive chat, Flash for the cron jobs.

The agent has eight tools:

  • ListActions, CreateAction, UpdateAction — read and mutate the plan;
  • CreateGoal — open a new workspace from a chat phrase like "quero começar a tratar minha saúde mental";
  • RememberFact, RecallFacts — long-term memory that survives across conversations;
  • WebSearch, WebFetch — Tavily-powered when I need fresh data.

Multi-tenant isolation is enforced by Eloquent global scopes on every user-owned model. Each user gets their own plan, memories, goals, and conversations — bulletproof from the start, because the alternative is a leak waiting to happen.

Goals as workspaces

The first version had one big plan and one long conversation. After a few weeks the conversation was a mess — health questions tangled with finance questions, the model couldn't decide which persona to wear, the plan flyout listed actions from contexts that had nothing to do with each other.

The fix was to model goals as workspaces: each goal has a sidebar entry, owns its own conversation history, owns its own actions in the plan, and tells the agent how to behave. A label like finance, fitness, emotional, learning picks the specialization prompt — finance gets hard math, emotional gets empathy-first, fitness gets consistency-over-intensity. general means "no specialization yet, ask me where to focus".

The agent itself can create goals via the CreateGoal tool. If you say "I want to start tracking my running", the agent confirms, calls CreateGoal(name: "Running", label: "fitness"), and the goal appears in the sidebar without you touching the UI. The whole loop — from intent to a new clean workspace — is conversational.

The hardest bug

The single most painful issue I shipped was an agent narrating an action it never executed. The user would say "adiciona pilates ao meu plano", the agent would reply "feito, adicionei o pilates, aqui está seu plano atualizado:", the response would cut off after the colon, and the action would never appear because CreateAction never fired. The model hallucinated the verb.

The fix was three layers:

  1. Detect — a decorateAssistantResponse() pass scans the final reply for preterite verbs (criei, adicionei, marquei, concluí) and checks whether the matching tool actually fired in this turn. If not, it appends a small italic warning and logs the conversation id at WARN level for postmortem.
  2. Auto-retry — when the first pass looks broken, Coach.php automatically runs a second stream pass with a corrective system nudge ("execute now, don't narrate"). Same conversation, so the model sees its own broken reply and the order to fix it. Pro almost always nails the second attempt.
  3. Reinforce the prompt — explicit ❌ / ✅ patterns showing the exact failure mode, plus a closing rule that forbids ending a response with a colon unless the list either appears in the text or a tool was already called.

I also moved from gemini-2.5-flash to gemini-2.5-pro for the interactive chat. Pro is slower (4-5s to first token versus 1-2s) but ~10x more reliable on multi-tool turns. The cron jobs stay on Flash because they're single-shot and short.

The polish layer

The core flow worked, but the app felt cheap. A few weeks of small UX commits later it stopped feeling cheap:

  • Pulsing dots in the assistant bubble as soon as you click Send — fills the 1-3s gap before Gemini emits its first token. The bubble appears in 88ms instead of staying empty for 1300ms.
  • Optimistic user-message insert — your message hits the thread in 18ms via Alpine, before the Livewire round-trip completes. A morph hook removes the placeholder when the server-rendered version arrives, no flicker, no duplicate.
  • Active-goal indicator in the sidebar — a 3px orange left-border + bg tint + bold title. You always know which workspace you're in.
  • Compacted composer — the FilePond drop-zone collapses to a small clip-icon button when no files are attached, freeing 130 vertical pixels for the chat.
  • Mobile/PWA — viewport locked against iOS pinch-zoom + URL-bar wobble, dynamic viewport units, overscroll containment, scroll-lock when overlays open.
  • Brand consistency — every primary CTA uses the orange accent; the green is reserved for "complete an action" where it carries semantic meaning.

What's interesting under the hood

Email replies thread the conversation. Every scheduled ping carries a Reply-To with subaddressing — reply+<conversationId>@coach.allur.co. When you reply from your phone, Resend's inbound webhook hits /webhooks/coach-email; the controller extracts the UUID, logs you in via auth()->login() so global scopes activate, and continues the same conversation. The agent sees the email body as if you typed it in the chat.

Memory has structure. CoachMemory stores facts with a kind (perfil, fatura, contrato, etc.) and a label. The agent saves proactively after analyzing a PDF or making a decision, so two months later "como estava aquela Santander de maio?" still works without the original document.

i18n is real. Every user-facing string is in lang/{pt_BR,en}/. Tests pass in both locales — CI runs with APP_LOCALE=en while local dev runs pt_BR, and any string asserted in Portuguese would fail in CI.

178 tests. TDD on every new feature: tool date-parsing, goal-scoped action creation, multi-tenant isolation (Alice never sees Bob's plan), invitation token validation, Reply-To routing, webhook auth, response-decorator detection. Pest 4, runs in <5 seconds.

Stack

  • Laravel 13 + PHP 8.4
  • Filament v5 for the admin panel and chat page
  • Livewire 4 with token streaming
  • laravel/ai SDK + Gemini 2.5 (Pro for interactive, Flash for cron)
  • Resend for transactional email and inbound replies
  • Tailwind v4 + a hand-tuned coach.css for the orange-accent minimalist look
  • Pest 4 for tests
  • SQLite locally, MySQL in production
  • Forge + Cloudflare for hosting

What's next

  • Per-goal custom specialization prompts (editable from the UI, not just the seven built-in labels)
  • An auto-switch tool — when the agent creates a goal mid-conversation, ask the user if they want to move there
  • Per-goal weekly briefings instead of one global recap
  • A small mobile-only tweak to the plan flyout for very long action titles

If any of this is useful to you, the repo is at github.com/allurco/coach — MIT, self-host, bring your own Gemini key. It's been stable in daily use as my own coach for a while now; I've shipped maybe 30 features-and-fixes since the first commit and the test suite is green every time.

Original use case: financial recovery. Now: anything you'd want a quiet, persistent, sometimes-too-honest accountability buddy for.
